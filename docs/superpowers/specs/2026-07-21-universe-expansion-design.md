# [Design Spec] 유동성 필터 기반 유니버스 확장 및 수집 자동화 시스템

* **작성일**: 2026-07-21
* **작성자**: BSenior AI Engineer
* **상태**: 승인 완료 (구현 진행 예정, Git 커밋 제외)

---

## 1. 개요 및 목적

현재 트레이딩 서비스의 ML 모델은 국내/해외 주식 및 코인 시장의 대표적인 대형주 위주(주식 90개, 코인 30개 등)로만 학습되고 있습니다. 이로 인해 챗봇의 종목 추천 다양성이 제한되고 상세 분석 페이지에서 서비스할 수 있는 대상 종목이 적다는 한계가 존재합니다.

본 설계는 **시장의 대표 지수 구성 종목**과 **거래대금/유동성 필터**를 조합하여 모델 학습 유니버스를 안전하게 확장하고, 확장 시 우려되는 API 호출 차단(Rate Limit), 신규 종목의 시계열 결측치 발생, 중소형주 거래량 부족에 따른 백테스트 착시(슬리피지) 문제를 기술적으로 해결하는 것을 목적으로 합니다.

---

## 2. 목표 사양 (Target Specifications)

* **대상 유니버스 규모**:
  * **국내 주식**: KOSPI 200 & KOSDAQ 150 편입 종목 중, 최근 60일 일평균 거래대금 50억 원 이상 (약 150~200개 종목)
  * **해외 주식**: S&P 500 편입 종목 중, 최근 60일 일평균 거래대금 1,000만 달러 이상 (약 200~300개 종목)
  * **코인**: 바이낸스(Binance) USDT 마켓 중, 일평균 거래대금 100만 달러 이상 및 상장 1년 이상 경과 (약 50~80개 종목)
* **네트워크/API 안정성**: 배치 수집 및 비동기 지연을 통해 yfinance 및 거래소 API 블락(Block) 발생 0건 달성
* **데이터 무결성**: 캔들 데이터 누락 시 전방 채움(`ffill`) 보정을 통해 전처리 단계에서의 훈련 데이터 드롭(Drop) 손실률 5% 이하 제어
* **백테스트 신뢰성**: 거래대금 크기에 반비례하는 슬리피지(Slippage) 페널티를 동적으로 부과하여 백테스트 수익률의 현실화

---

## 3. 상세 아키텍처 및 핵심 컴포넌트

### ① 유니버스 동적 스크리너 (Universe Dynamic Screener)
* **위치**: `ml/src/universe_screener.py` (신설 예정)
* **동작 매커니즘**:
  1. 외부 오픈 API 혹은 증권사 API를 통해 지수 구성원(KOSPI 200, KOSDAQ 150, S&P 500 등)과 바이낸스 전체 마켓 티커 리스트를 주기적으로 가져옵니다.
  2. 최근 60일 거래일의 시세 데이터를 조회하여 일평균 거래대금(시가 $\times$ 거래량)을 계산합니다.
  3. 최소 유동성 하한선(주식 50억 원 / 1,000만 달러, 코인 100만 달러)을 넘는 종목들만 필터링하여 유니버스 파일(`ml/configs/active_universe.json`)로 저장합니다.
* **주기**: 매주 일요일 밤 자동 실행하여 유니버스를 갱신합니다.

### ② 벌크 및 비동기 분할 수집 파이프라인 (Bulk & Async Data Pipeline)
* **위치**: `ml/src/data_collector.py` (기존 수집 스크립트 수정)
* **동작 매커니즘**:
  1. `active_universe.json`에서 대상 종목 리스트를 읽어옵니다.
  2. 단일 종목 루프 대신 `yfinance.download(tickers_list)` 벌크 요청을 사용하여 네트워크 오버헤드를 줄입니다.
  3. API 차단을 회피하기 위해 대상 종목 리스트를 30개 단위의 청크(Chunk)로 나눕니다.
  4. 청크 단위 수집 후 `time.sleep(2.0)`의 지연을 보장하여 초당 요청 한도(Rate Limit)를 엄격히 준수합니다.
  5. 네트워크 타임아웃 또는 연결 유실 시 최대 3회까지 지수 백오프(Exponential Backoff: 2초, 4초, 8초) 재시도를 적용합니다.

### ③ 피처 전처리 및 결측치 보정기 (Feature Preprocessor)
* **위치**: `ml/src/build_features.py` 및 `ml/src/preprocessing.py`
* **동작 매커니즘**:
  1. **상장 기간 검증**: 각 종목의 최초 데이터 시작일을 검증하여 상장 이력이 2년(730일) 미만인 종목은 원천 배제합니다.
  2. **결측 보정**: 거래정지나 통신 유실 등으로 캔들 가격이 비어있는 경우, `df.groupby('symbol').ffill()`을 적용해 이전 종가를 복사하여 채웁니다.
  3. **나머지 예외 처리**: 기술 지표 연산 중 `NaN`이 남는 경우(예: 볼린저 밴드 초기 구간 등), 해당 종목의 섹터 평균값 또는 0으로 종결 채움(`fillna`) 처리하여 학습 행이 탈락하지 않도록 보호합니다.

### ④ 슬리피지 반영 백테스트 (Slippage-aware Backtest Engine)
* **위치**: `ml/src/backtest_signals.py`
* **동작 매커니즘**:
  * 기존 백테스트 코드의 수수료 모델에 추가 슬리피지를 가변식으로 적용합니다.
  * **수식**:
    $$\text{Slippage Penalty (\%)} = \text{Base Slippage (0.05\%)} + \frac{\alpha}{\text{Median Dollar Volume (20d)}}$$
    * $\alpha$는 자산군별 변동성 비례 상수입니다.
    * 거래대금이 작고 호가가 얇은 종목일수록 슬리피지 비용이 크게 산정되어 백테스트 상에서 보수적인 수익률을 제공합니다.

## 4. 예외 처리 및 제약사항

* **상장폐지 대비**: 유니버스 스크리너 구동 시 관리종목이나 환기종목 태그를 조회하여 예외 없이 유니버스에서 즉시 제거합니다.
* **주말 수집 예외**: 주식 시장은 주말에 데이터가 갱신되지 않으므로, 코인 수집 배치와 주식 수집 배치를 독립된 스케줄로 분리 운영합니다.
* **로컬 메모리 압박**: 500개 종목 학습 시 메모리 누수가 발생하지 않도록 LightGBM 학습 루프 종료 시 가비지 컬렉터(`gc.collect()`)를 명시적으로 호출합니다.
