# AI 위탁운용 운영 콘솔 설계

상태: 대체됨. 심볼 기반 DCA·그리드 중심 설계는 사용자 요구인 시장 전체 자동선별과 맞지 않아 `2026-07-22-ai-fund-stock-selection-design.md`의 통합 자동선별 설계로 대체됐다.

- 작성일: 2026-07-22
- 상태: 구현 진행
- 범위: 기존 관리자 AI 위탁운용 화면에서 실제 운용 기능을 조작·검토할 수 있게 연결

## 목표

운영자가 별도 API 호출이나 DB 조작 없이 한 화면에서 설정을 저장하고, DCA·그리드 전략을 관리하며, 대기 주문 의도를 승인 또는 반려하고, 백테스트·포트폴리오·운영 이벤트를 확인한다.

## 화면 구조

기존 상단의 운용 상태와 시작·중지 제어는 유지한다. 그 아래를 다음 네 영역으로 구성한다.

| 영역 | 사용자 행동 | 데이터 |
| --- | --- | --- |
| 운용 설정 | 운용 모드, 자금, 위험 성향, 목표 비중 저장 | `admin_ai_fund_configs` |
| 전략 | DCA·그리드 생성, 실행·일시정지, 백테스트 실행 | `ai_fund_strategies`, `ai_fund_strategy_backtests` |
| 승인 대기 | 주문 근거·금액 확인, 승인·반려 | `ai_fund_trade_intents` |
| 포트폴리오·운영 | 성과·포지션·장애 이벤트 확인, 중단 설정 재개 | 원장, `ai_fund_operation_events` |

## API 계약

- `GET/POST /api/admin/ai-fund/strategies`: 사용자·거래소별 전략 목록 조회와 생성
- `POST /api/admin/ai-fund/strategies/{strategy_id}/status`: `RUNNING`, `PAUSED`, `HALTED` 상태 변경
- `GET /api/admin/ai-fund/intents`: 사용자·거래소·상태별 주문 의도 조회
- 기존 승인·반려, 백테스트, 성과, 운영 이벤트·재개 API를 화면에서 호출

전략 생성은 DCA 또는 GRID 필수 설정을 서버에서 검증한다. 모든 전략·리밸런싱 주문은 `PENDING` TradeIntent로 남고, 화면에서 승인된 경우에만 실행 경로에 진입한다.

## 오류와 상태

- API 호출 중에는 해당 버튼만 비활성화한다.
- 실패 이유는 해당 영역의 인라인 메시지로 보여 준다.
- 비어 있는 목록에는 다음 행동을 안내하는 빈 상태를 보여 준다.
- 기존 관리자 역할 검증 보류 정책은 유지하며, 현재 세션 Bearer 토큰으로 기존 API 계약을 호출한다.

## 검증

- Flask 라우트 테스트로 전략 목록·생성·상태 변경·의도 목록을 검증한다.
- Vitest로 운영 콘솔의 핵심 섹션과 승인 버튼 노출을 검증한다.
- 프론트엔드 빌드와 관련 백엔드 회귀 테스트를 수행한다.
