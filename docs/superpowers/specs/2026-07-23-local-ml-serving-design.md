# 로컬 ML 서빙과 AWS 거래 분리 설계

## 목표

로컬 개발 머신에서 코인 v10 및 주식 모델의 데이터 수집, 예측 생성, 재학습을 수행하고, AWS는 검증된 최신 예측을 이용한 매수와 보유 포지션 보호 매도만 수행한다.

## 운영 경계

### 로컬 ML 러너

- 코인: 30분마다 신규 캔들을 수집하고 현재 서빙 모델로 예측 CSV를 생성한다.
- 코인: 주 1회 재학습을 실행하고, 성능 검증을 통과한 모델만 서빙 모델로 승격한다.
- 국내 및 미국 주식: 각 시장 장 마감 이후 예측을 생성하고, 주 1회 재학습을 순차 실행한다.
- 생성물은 모델 버전, 생성 시각, SHA-256 해시를 포함한 매니페스트와 함께 AWS로 업로드한다.
- 이전 생성물을 덮어쓰기 전에 로컬 검증을 마친다. 실패한 수집·예측·학습 결과는 배포하지 않는다.

### AWS 거래 워커

- AI 위탁 신규 매수는 최신 예측 매니페스트가 유효할 때만 실행한다.
- 예측이 없거나 오래되었거나 매니페스트 검증에 실패하면 신규 매수를 보류한다.
- 보유 포지션의 익절, 손절, 비상정지, 주문 상태 동기화, 잔고·체결 원장 반영은 예측 유효성과 관계없이 계속 실행한다.
- 거래 워커는 학습·예측 생성 작업을 실행하지 않는다.

## 예측 유효성 정책

- 코인 신규 매수: 마지막 예측 생성 시각이 90분 이내일 때만 허용한다.
- 국내 주식 신규 매수: 가장 최근 국내 거래일 장 마감 뒤 생성된 예측만 허용한다.
- 미국 주식 신규 매수: 가장 최근 미국 거래일 장 마감 뒤 생성된 예측만 허용한다.
- 유효성 판정은 CSV 내부 행 시각이 아니라 배포 매니페스트의 UTC 생성 시각과 파일 해시를 기준으로 한다.
- 유효하지 않은 예측은 매수만 막으며, 보호 매도에는 적용하지 않는다.

## 배포 흐름

1. 로컬 러너가 임시 디렉터리에 모델, 예측 CSV, 매니페스트를 생성한다.
2. 로컬 검증이 모델 버전, 예측 행 수, 최신 시각, 파일 해시를 확인한다.
3. 검증이 성공하면 SSH/rsync로 AWS 임시 릴리스 디렉터리에 업로드한다.
4. AWS 검증 스크립트가 해시와 매니페스트를 재검증한다.
5. AWS에서 자산별 현재 릴리스 심볼릭 링크(`current/crypto`, `current/kr_stock`, `current/us_stock`)를 새 릴리스로 원자적으로 교체한다.
6. 거래 워커는 해당 자산의 현재 릴리스 링크만 읽는다. 불완전한 업로드 파일은 읽지 않는다.

## 장애 처리

- 로컬 머신이 꺼지거나 절전 상태이면 다음 예측 배포가 생략된다.
- AWS는 마지막 유효 예측의 허용 시간이 지나면 신규 매수를 자동 보류한다.
- 보유 자산의 보호 매도 스케줄러와 비상정지는 AWS에서 계속 동작한다.
- 로컬 학습 실패, 전송 실패, AWS 해시 검증 실패는 기존 릴리스를 유지하고 실패 원인을 운영 로그와 대시보드 상태에 기록한다.

## 범위 제외

- AWS EC2에서 v10 전체 재학습을 실행하지 않는다.
- 예측이 오래된 경우 신규 매수 차단을 제거하지 않는다.
- 로컬 머신이 꺼진 상태에서 새 매수 신호를 생성하지 않는다.

## 검증 기준

- 로컬 러너가 코인 예측 릴리스를 생성하면 AWS의 현재 릴리스가 원자적으로 갱신된다.
- 해시가 일치하지 않는 릴리스는 AWS가 거부한다.
- 예측이 만료된 상태에서는 신규 매수가 발생하지 않는다.
- 예측이 만료된 상태에서도 조건매매 보호 매도와 비상정지는 실행 가능하다.
- 코인, 국내 주식, 미국 주식의 유효성 기준이 각각 독립적으로 검증된다.
