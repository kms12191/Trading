# AI 위탁운용 통합 자동선별 설계

상태: 2026-07-22 구현 및 원격 DB 마이그레이션 적용 완료. 토스 외부 API의 DNS·인증 상태는 별도 운영 준비 항목이다.

## 목적

사용자가 종목 코드나 심볼을 지정하지 않고, 코인원·바이낸스의 가상자산과 토스증권의 국내·미국 주식 활성 ML 예측 결과에서 매수 후보를 자동 선별해 운용한다. 사용자는 운용 자금, 거래소, 주식 시장 범위, 위험도, 최대 보유 종목 수, 모의·실거래 모드만 설정한다.

## 검토한 방식

1. 심볼 지정 DCA·그리드: 특정 종목을 정기·구간 매매하는 도구다. 종목 선택을 자동화하지 못하므로 본 기능의 주 화면으로 사용하지 않는다.
2. 예측 점수 상위 1개 직접 매수: 구현은 짧지만 국내·미국 시장을 함께 운용할 때 집중 위험이 크고, 기존 보유 종목과의 중복을 충분히 제어하기 어렵다.
3. 자산군별 후보군 선별 후 위험 한도 내 다종목 운용: 기존 코인 ML 선별은 유지하고, 국내·미국 활성 모델의 LONG 신호를 추가한다. 정책 통과·신선도·점수·기보유 종목을 기준으로 후보를 고른 뒤 최대 보유 종목 수와 종목별 한도를 적용한다. 이 방식을 채택한다.

## 구성

### 1. 통합 후보 선별

- 코인원·바이낸스는 현재의 `crypto_predictions_lgbm_v10.csv` 기반 자동선별 경로를 유지한다.
- 토스는 주식 전용 후보 선별 서비스를 사용한다. 두 경로 모두 심볼을 사용자가 입력하지 않고 ML 결과에서 얻는다.

- `STOCK_KR`, `STOCK_US` 활성 모델의 최신 예측 파일을 읽는다.
- `LONG`, 정책 미차단, 양수 신호점수, 최신 예측만 후보로 인정한다.
- 설정한 시장 범위(`KR`, `US`, `ALL`)에 따라 후보를 합친다.
- 이미 보유한 종목은 신규 진입 후보에서 제외한다.
- 신호점수 내림차순으로 정렬하고, 시장별 목표 비중과 최대 보유 종목 수를 넘지 않는 후보만 반환한다.
- 후보에는 `market`, `symbol`, `confidence_score`, `signal_id`, `model_version`, `selection_reason`을 남겨 주문·감사 추적에 사용한다.

### 2. 운용 설정

`admin_ai_fund_configs`의 토스 설정에 다음 값을 추가한다. 코인원·바이낸스는 기존 설정만으로 동작한다.

- `asset_scope`: `KR`, `US`, `ALL` 중 하나, 기본 `ALL`
- `max_open_positions`: 동시에 보유 가능한 종목 수, 기본 3
- `kr_allocation_pct`, `us_allocation_pct`: 시장별 신규 진입 자금 배분. 합계는 100이다.
- `selection_refresh_minutes`: 동일 예측 신호 재평가 주기. 기본 60분이다.

기존 `allocated_capital`, `max_position_size`, `risk_preset`, `min_signal_confidence`, 손절·익절 정책과 운용 모드를 그대로 적용한다.

### 3. 스케줄러 흐름

1. 주문 대사, 회로 차단기, 승인된 주문 의도 실행, 보유 종목 청산 검사를 수행한다.
2. 코인원·바이낸스 설정이면 기존 코인 CSV 자동선별을 수행하고, 토스 설정이면 주식 후보 선별 서비스를 호출한다.
3. 종목별 현재가를 조회하고 기존 `AdminAiManagedTrader` 위험 검사를 통과한 종목만 주문한다.
4. 모의 모드에서는 모의 체결을 기록하고, 실거래·카나리 모드에서는 토스 주문 클라이언트를 사용한다.
5. 예측 파일·토큰·현재가 중 하나라도 준비되지 않으면 주문하지 않고 운영 이벤트에 실패 원인을 남긴다.

### 4. 화면

- 기존 `AdminAiFundDashboard`를 확장한다. 새 운용 화면으로 교체하지 않는다.
- 거래소 선택은 코인원·바이낸스의 `가상자산 자동선별`, 토스의 `주식 자동선별` 범위를 함께 설명한다.
- 토스 주식 설정에는 시장 범위, 시장별 배분, 최대 보유 종목 수, 위험도와 운용금만 노출한다.
- "현재 AI 후보" 표에 국내·미국 후보, 점수, 모델, 선별 사유를 표시한다. 종목을 직접 입력하는 칸은 없다.
- 후보가 없으면 시장별 `POLICY_BLOCKED` 또는 `NO_LONG_SIGNAL` 상태와 모델의 시장 국면을 표시한다. 빈 후보를 오류나 임의의 종목 추천으로 표현하지 않는다.
- 심볼 기반 DCA·그리드는 자동선별 주 화면에서 제거한다. 필요하면 이후 암호자산 전용 고급 전략으로 별도 진입점에 둔다.

## 실패 처리와 운영 원칙

- 토스 API DNS·인증·현재가 오류는 재시도 후 해당 주기의 신규 진입을 중단한다.
- 청산 조건은 신규 후보 선별 실패와 독립적으로 계속 평가한다.
- 실거래는 `REAL` 또는 `CANARY` 모드와 유효한 사용자 토스 자격증명이 모두 있을 때만 가능하다.
- 활성 주식 모델이나 최신 예측 파일이 없으면 자동매매를 시작하지 않고 화면에 준비 상태를 알린다.

## 검증

- 후보 선별: 시장 범위, 정책 차단, 점수 기준, 중복 보유, 최대 보유 수, 시장 배분을 단위 테스트한다.
- 스케줄러: 코인원·바이낸스는 기존 코인 신호를 유지하고, 토스 설정만 주식 후보 선별을 사용하는지 검증한다.
- API: 자동선별 후보와 설정 검증 응답을 테스트한다.
- 프론트엔드: 심볼 입력 없이 주식 자동선별 설정 payload가 만들어지는지 모델 테스트한다.

## 범위 밖
