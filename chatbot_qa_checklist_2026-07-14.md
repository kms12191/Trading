# 챗봇 QA 체크리스트

- 기준일: 2026-07-14
- 범위: 챗봇 화면, 시세/계좌 조회, 뉴스/공시/RAG, 매매 제안/승인 안전 정책
- 담당: A/B/C/D 각 25개, 총 100개
- 체크 방법: 각 항목의 예상 질문을 챗봇에 입력하고, 기대 답변과 확인 포인트를 기준으로 `PASS`, `FAIL`, `BLOCKED` 중 하나를 기록한다.

| 상태 | 의미 |
|---|---|
| PASS | 기대 답변과 확인 포인트를 모두 만족 |
| FAIL | 응답 오류, UI 깨짐, 안전 정책 위반, 원문 에러 노출 등 결함 확인 |
| BLOCKED | API 키/권한/외부 API/환경 문제로 검증 불가 |

## QA 기록 규칙

- 실패 항목은 `비고`에 재현 경로, 입력 문장, 실제 응답, 콘솔/네트워크 오류 여부를 기록한다.
- 거래소 API나 외부 뉴스 API 장애로 검증이 막힌 경우 `BLOCKED`로 표시하고, 장애 원인을 비고에 남긴다.
- 보안 관련 실패는 우선순위를 높게 둔다. 특히 API secret, raw exception, stack trace, 거래소 raw payload가 사용자 화면에 노출되면 즉시 수정 대상으로 분류한다.
- 매매 관련 항목에서 승인 전 주문 실행이 확인되면 배포 차단 결함으로 분류한다.
